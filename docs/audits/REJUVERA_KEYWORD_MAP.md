# REJUVERA — Keyword Map & Topical Strategy

> **تنويه:** لا وصول إلى Search Console/Ads → **لا بيانات ترتيب/حجم/CTR**. هذه **استراتيجية مقترحة** مبنية على نية البحث والمحتوى القائم، وليست بيانات أداء. لا أرقام زيارات/مبيعات مُختلقة.

## الوضع القائم (مؤكد)
- 5 خدمات «أساسية» مُعلاة بكلمات/عناوين AR+EN جاهزة في `data/core-services.json`: شد الوجه، شد الرقبة، الوذمة الشحمية، تجميل المهبل، تضييق بعد الولادة.
- **40+ خدمة** متاحة في قائمة `/contact` (دليل حي) → الكتالوج أوسع من الـ5؛ الفجوة في **التحديد كأولوية + عمق صفحات الإجراء + الربط الداخلي**، لا في الغياب الكامل.
- بنية URL: `/services/[slug]` (ASCII) + `/doctors/[slug]` + `/journal/[slug]` + `/p/[slug]` للحملات.
- schema: `MedicalBusiness` عام؛ **ينقص** `MedicalProcedure`/`Physician`/`BreadcrumbList`/`FAQPage` على مستوى الصفحة.

## البنية المستهدفة (Topical Authority)
```
Service Hub (/services)
 → Category (عمليات تجميل الوجه / جراحة التجميل / التجميل النسائي / نحت الجسم / الثدي ...)
   → Procedure Page (/services/[slug])  ← MedicalProcedure + FAQPage + breadcrumb
     → Doctor Page (/doctors/[slug])    ← Physician schema، ربط متبادل
     → Supporting Journal/FAQ (/journal/[slug])
       → Consultation CTA (نموذج inline + WhatsApp + Call)
```

## خريطة الكلمات حسب المجموعة
> الأعمدة: نية | مرحلة | صفحة حالية | إجراء مقترح | cannibalization | schema | أولوية.
> «صفحة حالية = أساسية» تعني مؤكّدة في core-services.json. «؟» = تتطلب تحقّق وجود/عمق الصفحة.

### الوجه والرقبة
| كلمة رئيسية | نية | مرحلة | صفحة حالية | إجراء | schema | أولوية |
|---|---|---|---|---|---|---|
| شد الوجه في الرياض | تجارية/طبية | قرار | أساسية `/services/facelift-surgery` | عمّق: FAQ، نتائج، طبيب، نموذج inline | MedicalProcedure+FAQ | P1 |
| شد الرقبة في الرياض | تجارية | قرار | أساسية `/services/neck-lift-surgery` | نفس أعلاه | MedicalProcedure+FAQ | P1 |
| شد الوجه والرقبة / deep plane facelift | تجارية/معلوماتية | بحث-قرار | جزئي | صفحة/قسم يربط الاثنين + مقال «deep plane» | Article+FAQ | P2 |

> **تحذير cannibalization:** لا تُنشأ صفحة «شد الوجه والرقبة» منفصلة تنافس صفحتي الوجه/الرقبة على نفس النية؛ اربطها كمحتوى داعم/قسم، أو اجعلها صفحة مظلّة واضحة الاختلاف.

## قواعد التنفيذ (تجنّب الضرر)
1. **لا تكرار نية**: قبل إنشاء صفحة، تأكّد ألا تنافس صفحة قائمة على نفس النية (افحص dropdown الـ40+ والـ`/services/[slug]` الحية).
2. **عمق قبل اتساع**: عمّق الخدمات الأساسية الـ5 (FAQ/نتائج/طبيب/نموذج inline) قبل توليد صفحات جديدة.
3. **ربط داخلي**: كل صفحة إجراء ↔ طبيب ↔ مقال داعم ↔ CTA استشارة.
4. **schema بصدق**: FAQPage فقط حين يكون الـ FAQ ظاهرًا فعليًا؛ لا review schema بلا تقييمات حقيقية.
5. **محتوى محلي حقيقي** (الرياض، أحياء، خرائط) بلا حشو كلمات.

## غير مؤكد (يتطلب وصولًا)
- أيّ الكلمات تجلب ظهورًا/نقرات فعليًا، ومواضع 4–20، وأي cannibalization فعلي → **يتطلب GSC**.
- المطابقة الدقيقة dropdown(40+) ↔ صفحات `/services/[slug]` الموجودة فعلًا → يتطلب DB/زحف كامل.
