## ملاحظات الجودة (مؤكدة)

### إيجابيات
- **صياغة طبية متوافقة**: «تقييم»، «خيارات مناسبة»، «نتائج واقعية»، «دون وعود موحّدة لكل الحالات» (`core-services.json:110-111,154`) — تتجنّب الضمانات/الادعاءات الممنوعة.
- **ثنائية AR+EN منظّمة** لكل خدمة (عناوين/أوصاف/كلمات/منافع) — أساس محتوى قوي.
- **NAP متّسق** عبر الموقع والإعدادات والschema (هاتف 0114999959 / 9200 17403، بريد info@rejuvera.sa، ساعات سبت-خميس 2-10م = `openingHours 14:00-22:00`).

### للتحسين / مخاطر
- **CONTENT-001 (Medium)** ازدواج نص AR+EN داخل نفس DOM (تبديل CSS) → نص مكرر في الصفحة + ثقل (مرتبط PERF-005/SEO-001). قرار اللغة (SEO-001) يحسم هذا.
- **CONTENT-002 (Low-Med)** تكرار اسم «د. لؤي السالمي» في عناوين SEO لعدة خدمات (`core-services.json:31,80,124`) → نمط عناوين متقارب؛ مقصود كسلطة لكن راقب التكرار المفرط.
- **CONTENT-003 (Verify)** صفحة الخدمة بلا FAQ/نتائج/معالجة اعتراضات (حي) → محتوى تحويلي ناقص (CRO).
- **CONTENT-004 (Verify)** خطر صفحات رقيقة/متقاربة ضمن الـ40+ خدمة وصفحات `/p/[slug]` — يلزم زحف لكشف thin/duplicate.
- **CONTENT-005 (Info)** الصور الأساسية SVG/curated عامة (`/media/curated/...`) لبعض الخدمات → استبدال بصور حقيقية للمركز يرفع الثقة (مع alt وصفية).

## قائمة «تحتاج مراجعة طبية» (منفصلة، لا تُعدَّل بلا مختص)
- تجميل المهبل، تضييق بعد الولادة، الوذمة الشحمية، وأي محتوى يصف نتائج/مخاطر إجراء جراحي.
- أي إضافة FAQ طبية مستقبلية يجب أن تُراجَع طبيًا قبل النشر وقبل إضافة FAQ schema.

## جرد تفصيلي مطلوب (لاحقًا، يتطلب وصولًا)
1. زحف كامل لـ `/services/[slug]`,`/journal/[slug]`,`/p/[slug]` → جدول: URL، عنوان، طول، نية، تصنيف (Keep/Improve/Merge/Redirect/Remove/Noindex/Thin/Duplicate).
2. كشف العناوين/الأوصاف المكرّرة عبر الزحف.
3. مطابقة أسماء الخدمات/الأطباء/الأرقام عبر كل الصفحات (تناسق).
4. مراجعة لغوية AR (إملاء/مصطلح) وEN (ترجمة حرفية) لكل صفحة.
